dns prefetch

DNS Prefetch 是一种DNS 预解析技术，在浏览网页时，浏览器会在加载网页时对网页中的域名进行解析缓存，这样在点击当前网页中的链接或者加载静态资源时就无需进行DNS的解析，减少用户等待时间，提高用户体验。

DNS解析需要用域名去匹配IP，这个是需要时间的，如果加了dns-perfetch，浏览器会记住（缓存）这个解析，直接请求，不需要再走DNS。
<link rel="dns-prefetch" href="//cdn.www.sojson.com">
打开DNS 预解析
<meta http-equiv="x-dns-prefetch-control" content="on">
关闭
<meta http-equiv="x-dns-prefetch-control" content="off">

preload与prefetch

preload与prefetch都是link元素中的rel属性值。
preload 让浏览器提前加载指定资源(加载后并不执行)，将加载和执行分离开， 使用preload后，不管资源是否使用都将提前加载。若不确定资源是必定会加载的，则不要错误使用preload，以免给页面带来更沉重的负担。对跨域的文件进行preload时，必须加上 crossorigin 属性
<link rel="preload" crossorigin href="..." as="...">
as 属性的作用是告诉浏览器被加载的是什么资源, 可能的 as 值包括：

  • script
  • style
  • image
  • media
  • font
  • document
    使用 as 来指定将要预加载的内容的类型，将使得浏览器能够：
  • 更精确地优化资源加载优先级
  • 匹配未来的加载需求，在适当的情况下，重复利用同一资源
  • 为资源应用正确的内容安全策略
  • 为资源设置正确的 Accept 请求头

prefetch的作用是告诉浏览器加载下一界面可能会用到的资源，因此该方法的加载优先级非常低。
<link rel="prefetch" href="./nextPage.js">
一般对于当前页面的资源使用preload，对于可能在将来的页面中使用的资源使用 prefetch
